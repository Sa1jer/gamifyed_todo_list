# Architecture Inventory

Last updated: 2026-07-15

This is the living inventory for incremental decomposition. The supported
baseline is Flutter `3.44.3` with Dart `3.12.2`.

## Layer Map

| Layer | Current owners |
|---|---|
| Startup/runtime facade | `lib/main.dart`, `lib/app_state.dart`, `lib/app_state/provider.dart` |
| Domain mutation policy | `lib/coordinators/` |
| Domain models | compatibility barrel `lib/models.dart`, declarations in `lib/models/` |
| Pure rules/read indexes | `lib/engines/`, `lib/analytics/` |
| Persistence orchestration | `lib/persistence/`, compatibility facade `lib/storage_service.dart` |
| Presentation data | `lib/presentation/` |
| Presentation | `lib/widgets/`, `lib/theme/` |
| Regression suite | `test/` |

`AppState` remains the public facade. Coordinators receive explicit mutable
entities/collections, never persist or notify, and return simple results.
`AppState` owns cross-domain ordering, device side effects, one persistence
request, and the final notification.

## Implemented Boundaries

### Read models

- `CompletionHistoryIndex` owns effective completion indexing.
- `AnalyticsReadModel` projects history and skills into scalar immutable
  records. It contains no Flutter types or live mutable model references.
- `WeeklyAnalyticsBuilder` accepts explicit inputs and returns
  `WeeklyAnalyticsViewData`; output records contain IDs, text, dates, enums,
  and numbers, not `AppState`, `Task`, `Skill`, or `WeeklyGoal` references.
- `TodayDashboardViewDataBuilder` and `TasksPanelViewData` remove repeated
  filtering/sorting from widget builds.
- Analytics cache invalidation is conservative: every public AppState
  notification advances the epoch. This favors correctness over avoiding an
  occasional rebuild after unrelated preference changes.
- `AppStateProvider` is an `InheritedNotifier`; the application shell uses an
  `AppStateSelector` for persistence/tooltips state, while existing feature
  widgets retain compatible broad observation.

### Domain mutation

- `TaskMutationCoordinator`: task CRUD, recurrence normalization, Inbox and
  Stage-link policy.
- `RoadmapMutationCoordinator`: Stage graph/path/template mutation and link
  cleanup.
- `TaskCompletionCoordinator`: normal, minimum-action and Inbox completion,
  XP/stat/history mutation, recurrence and undo result data.
- `RewardMutationCoordinator`: buffs, chests, source-key idempotency, claims,
  consumption and undo restoration.
- `SkillGoalMutationCoordinator`: skill/goal/RoadMap-cycle mutation and
  reference cleanup.
- `ReviewSessionCoordinator`: selection and review/session state policy.

### Persistence

- `SaveScheduler` owns debounce, single-flight writes, trailing requests,
  lifecycle flush, error propagation and disposal.
- `SnapshotStore` owns committed current/previous payload access.
- `StorageSnapshotCodec` owns snapshot serialization and decoding.
- `StorageMigrationPolicy` owns legacy/version compatibility decisions.
- `StorageService` remains the public compatibility facade; schema, manifest
  order, authoritative empty state, and failed-load write blocking are
  unchanged.

### Presentation

- Mobile journal/statistics sections, desktop statistics sections, selected
  skill header and desktop right rail are ordinary modules.
- Weekly Analytics uses shared view data plus extracted section/goal widgets.
- Task form text/focus/controllers and visual sections have separate owners.
- `shared.dart` remains a compatibility entry point while surfaces, buttons,
  progress/badges, dashed borders, form controls and motion controls live in
  cohesive modules.

## Remaining Risks

| Priority | Area | Evidence / next action |
|---|---|---|
| P1 | Feature-level broad AppState observation | `TasksPanel`, Today, RoadMap and parts of the desktop composition still rebuild from the facade; profile these boundaries before adding further selectors. |
| P1 | Desktop composition | `desktop_workspace.dart` remains a 2241-line `part`; extract sidebar/main composition with explicit inputs in a later focused batch. |
| P1 | Weekly Analytics shell | 1549 lines remain; mutations and navigation still live in the stateful shell. |
| P1 | Native memory evidence | Static allocation work is complete for touched paths, but repeated-route heap growth needs profile snapshots. |
| P2 | Domain model Flutter metadata | Skill/achievement models still carry `Color`/`IconData` for compatibility; no schema migration was attempted. |

## Non-Goals Preserved

No Hive/schema, snapshot semantics, XP, Goal, recurring, RoadMap, reward or
completion product rules changed. No state-management migration, package
upgrade, visual redesign or deletion of unproven legacy code occurred.
