explicit owners.

## Implemented Boundaries

- `lib/analytics/`: bounded scalar analytics snapshots and weekly view data.
- `lib/coordinators/`: six cohesive mutation coordinators behind AppState.
- `lib/persistence/`: save scheduling, snapshot store, codec and migration
  policy.
- `lib/models/`: eight declaration modules with the old import retained as a
  compatibility barrel.
- `lib/presentation/`: deterministic Today/Tasks/RoadMap preparation.
- `lib/widgets/weekly_analytics/`: reusable weekly sections and goal editor.
- `lib/widgets/shared/`: cohesive surface, button, progress, border, form and
  motion modules with compatibility exports.
- `lib/widgets/main_page/desktop_right_rail.dart`: ordinary module extracted
  from the desktop part library.

## Material Presentation Reductions

The program materially reduced mobile journal, desktop statistics, Today
Dashboard, TasksPanel, task dialog, Weekly Analytics, RoadMap inspector,
rewards/bosses, shared widgets and desktop workspace. The exact prior sizes are
available in Git history; current sizes are generated by
`dart run tool/architecture_audit.dart`.

## Remaining Large Shells

- `desktop_workspace.dart` remains a 2240-line `part` and still combines
  navigation, selection, responsive composition and dialog orchestration.
- `weekly_analytics_dialog.dart` remains a 1571-line shell after extracting
  repeated sections and scalar read data.
- `tasks_panel.dart` and several secondary dialogs remain sizeable.
- `storage_service.dart` still owns Hive box opening and compatibility reads;
  the extracted helpers deliberately did not rewrite recovery semantics.

These are evidence-backed P1 follow-ups, not hidden completed work.
