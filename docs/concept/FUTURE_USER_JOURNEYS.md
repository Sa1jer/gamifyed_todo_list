# Future User Journeys

These are **product hypotheses** for future validation. They use real current
entities where possible and avoid requiring a diagnostic label or a perfect
plan.

## 1. Large vague intention

| Element | Journey |
| --- | --- |
| User state | "I want to make a video, but I do not know what counts as starting." |
| Trigger | Creates or opens a Skill / intention. |
| Entry | Next Action Lens asks for one observable first result, not full RoadMap design. |
| System response | Suggests a Boot sequence: open project, assemble first 30 seconds of A-roll, watch once. |
| Decisions | Choose or edit one action; optional minimum action. |
| Success | A concrete artifact or observation exists. |
| Tone | "Start small enough to see something change." |
| Failure risk | User is asked to name too many stages before acting. |
| Remember | Chosen context, first result, next smallest move. |

## 2. Cannot start today

| Element | Journey |
| --- | --- |
| User state | Existing task feels disproportionately hard. |
| Trigger | Voluntary "Нужен другой вход" / Resistance signal. |
| Entry | Existing task remains intact; no forced new type. |
| System response | Offers: make smaller, clarify result, identify blocker, define stop point, or open Boot sequence. |
| Decisions | One intervention choice; dismiss remains valid. |
| Success | User performs one bounded entry action or deliberately pauses with context. |
| Tone | "The task may be too large for this moment; choose a smaller entry." |
| Failure risk | The app turns resistance into a lengthy self-assessment. |
| Remember | Optional selected intervention and later outcome, not a diagnosis. |

## 3. Productive rabbit hole

| Element | Journey |
| --- | --- |
| User state | Completes research or maintenance but Main Quest outcome does not move. |
| Trigger | Voluntary reflection or review, not surveillance. |
| Entry | Quest Chain / current intention view. |
| System response | Asks neutrally: "Did this move the result you chose?" Offers park, link as support work, or choose next Main action. |
| Decisions | Keep as useful support, park it, or reselect focus. |
| Success | Exploration is acknowledged without displacing the intended next result. |
| Tone | "Useful work can be support work; choose what you want to move next." |
| Failure risk | Side Quest labels shame legitimate discovery. |
| Remember | User's focus boundary, not a moral score. |

## 4. Started but became stuck

| Element | Journey |
| --- | --- |
| User state | The task is in progress, but the next move is unknown. |
| Trigger | User pauses or opens task after an interruption. |
| Entry | Current task details / Return Card. |
| System response | Shows completed evidence, optional blocker, and one next move; suggests splitting only if needed. |
| Decisions | Name blocker, choose a small next action, or stop with a Save Point. |
| Success | The work resumes without rebuilding the entire plan. |
| Tone | "You do not need to solve the whole task to name the next move." |
| Failure risk | Forced splitting creates more planning work. |
| Remember | Result, blocker, first return action, and useful links. |

## 5. Cannot finish

| Element | Journey |
| --- | --- |
| User state | Keeps polishing or expanding a nearly finished result. |
| Trigger | User selects Finish mode or marks the task as near completion. |
| Entry | Definition of Done prompt. |
| System response | Shows optional acceptable boundary and a Finish Quest such as "watch once and note one concrete issue". |
| Decisions | Accept, revise boundary, park improvement, or continue intentionally. |
| Success | A user-chosen adequate result is completed. |
| Tone | "What would make this useful enough to release?" |
| Failure risk | Premature closure or rigid standards. |
| Remember | Chosen boundary and parked improvements if user elects to keep them. |

## 6. Returning after several days

| Element | Journey |
| --- | --- |
| User state | Feels behind and cannot remember where work stopped. |
| Trigger | Opens a context after absence. |
| Entry | Return Card, not a giant backlog. |
| System response | "You paused after X. The next small move was Y." Offers resume, choose another action, or start fresh. |
| Decisions | One of three low-cost choices. |
| Success | Re-entry occurs without a streak-loss message or full project reconstruction. |
| Tone | "You are still on the journey; here is your last useful context." |
| Failure risk | Stale context is presented as an obligation. |
| Remember | Timestamp and user ability to dismiss outdated guidance. |

## 7. Low-energy day

| Element | Journey |
| --- | --- |
| User state | Wants continuity but not normal workload. |
| Trigger | Voluntary capacity choice. |
| Entry | Current focus or Inbox. |
| System response | Offers small valid connection: prepare environment, inspect result, make one correction, or explicitly rest. |
| Decisions | Select capacity and action, or leave without penalty. |
| Success | The user retains agency and context without pretending to complete normal work. |
| Tone | "Choose an entry that fits this moment." |
| Failure risk | Capacity labels become self-judgment or medical inference. |
| Remember | Nothing by default; persistence must be user-controlled. |
